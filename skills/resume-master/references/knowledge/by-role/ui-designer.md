# UI/UX 设计师简历怎么写：作品集展示 + 设计思维的简历模板

> 本文为 UI/UX 设计师提供一份从作品集整合到设计思维表达的完整简历写作指南。解决设计师简历中"重视觉轻逻辑、有作品无数据、设计决策说不清"三大核心问题，帮助你构建一份既能通过 ATS 筛选、又能打动设计主管的专业简历。适合交互设计师、视觉设计师、产品设计师及 UI/UX 全链路设计师阅读。

设计师的简历面临一个独特矛盾：**你需要用标准化的文字格式（ATS 友好），来展示本质上属于视觉化的能力（设计作品）。** 很多设计师因此走入两个极端——要么把简历做成炫酷的作品集（ATS 根本无法解析），要么写了一份毫无设计特色的纯文字简历（无法体现专业能力）。正确的做法是：**用结构化的文字简历通过 ATS 初筛，同时用作品集链接在面试环节展现设计实力。** 本文将拆解这套"双轨策略"的完整执行方法。

## 一、作品集与简历的关系：分工明确，各司其职

首先，明确一个核心认知：**简历 ≠ 作品集。** 它们是两个独立但互补的求职武器。

| 维度 | 简历（Resume） | 作品集（Portfolio） |
|------|---------------|-------------------|
| **核心目的** | 通过 ATS 筛选，获得面试机会 | 在面试中展示设计能力与思维深度 |
| **阅读对象** | ATS 系统 + HR（非设计专业） | 设计主管 + 团队面试官 |
| **内容重点** | 工作经历、技能关键词、量化成果 | 完整项目案例、设计过程、视觉呈现 |
| **格式要求** | 单栏文字、标准字体、PDF | 网页/PDF，自由排版，高质量截图 |

### 1.1 简历中如何引用作品集

在简历顶部的联系信息区域，添加作品集链接：

```
[你的姓名]
[电话] | [邮箱] | Portfolio: [你的作品集链接] | Dribbble: [链接]（可选）
```

**链接选择优先级**：
1. **个人网站**（如 yourname.com）— 最专业，可完全控制内容
2. **Behance / Dribbble** — 行业认可度高，曝光机会多
3. **Notion 作品集页面** — 灵活易更新，适合快速搭建
4. **PDF 作品集**（附在邮件中）— 备选方案，确保面试官一定能看到

**重要提醒**：确保作品集链接在投递时是可访问的（非私密权限），且加载速度流畅。一个打不开的作品集链接比没有更糟糕。

## 二、项目经历：用设计思维的语言描述你的工作

设计师简历中的项目描述，最常见的问题是"只说了做了什么界面，没说为什么这么做"。面试官想看到的是你的 **设计决策过程** 和 **业务影响**，而非像素级的视觉描述。

### 2.1 项目描述的黄金结构

每个项目应遵循 **"设计挑战 → 用户洞察 → 设计方案 → 量化结果"** 的逻辑链。

### 2.2 改前 vs 改后案例

**案例一：移动端电商 APP 改版**

> **改前（视觉描述、缺乏逻辑）：**
> - 负责电商 APP 首页和商品详情页的 UI 设计。
> - 使用 Figma 完成了高保真原型和设计规范。
> - 配合开发完成了设计还原。

> **改后（设计思维、业务驱动）：**
> - **设计挑战**：首页信息架构混乱，用户调研显示 **63%** 的用户表示"找不到想买的商品"，首页跳出率高达 **48%**。
> - **用户洞察**：通过 **用户访谈**（12 名目标用户）和 **热力图分析** 发现，用户注意力集中在页面前 30% 区域，但核心入口（分类导航）被推至页面中部。
> - **设计方案**：重构首页信息架构，将分类导航提升至首屏黄金位置；采用**卡片式布局**提升商品信息的扫视效率；建立 **8px 网格系统** 和组件化设计规范，统一 40+ 页面的视觉一致性。
> - **量化结果**：改版上线后，首页跳出率从 **48% 降至 31%**（降低 **35%**），商品详情页到达率提升 **27%**，设计规范使后续迭代效率提升 **40%**。

**案例二：SaaS 产品 Dashboard 设计**

> **改前（功能罗列、无用户视角）：**
> - 设计了后台管理系统的数据 Dashboard。
> - 包含图表、筛选器和导出功能。

> **改后（用户研究驱动、效率量化）：**
> - **设计挑战**：运营团队每日需花费 **90 分钟** 在多个系统间切换查看数据，信息获取效率极低。
> - **用户洞察**：通过 **影子观察法**（Shadowing）跟踪 5 名运营人员的日常操作流程，识别出 3 个高频数据查看场景和 7 个核心指标需求。
> - **设计方案**：设计"一站式"运营 Dashboard，运用 **信息层级理论** 将核心指标置于 F 型视觉动线的关键位置；引入**可定制卡片**机制，允许用户自由组合数据模块；设计**异常预警**视觉系统（红/黄/绿状态指示）。
> - **量化结果**：运营团队日均数据查看时间从 **90 分钟缩减至 25 分钟**（效率提升 **72%**），用户满意度评分（NPS）从 **32 提升至 78**。

### 2.3 设计师特有的量化维度

很多设计师觉得"设计很难量化"。其实，以下维度都可以作为量化指标：

- **用户行为指标**：点击率（CTR）、转化率、跳出率、完成率、任务成功率
- **效率指标**：操作步骤减少、任务完成时间缩短、开发还原效率提升
- **体验指标**：NPS（净推荐值）、SUS（系统可用性量表）评分、用户满意度
- **设计系统指标**：组件复用率、设计规范覆盖页面数、迭代效率提升比例
- **商业指标**：功能上线后的收入增长、注册转化提升、客户流失率下降

## 三、技能展示：工具熟练度 + 设计方法论

### 3.1 设计工具的分层展示

```
设计工具：
- 精通：Figma（3年主力工具，组件库搭建与 Auto Layout 深度使用）、Sketch
- 熟练：Adobe XD、Photoshop、Illustrator
- 了解：After Effects（微交互动效制作）、Principle、Framer

用户研究方法：
- 用户访谈、可用性测试、A/B 测试、问卷调查、卡片分类法、影子观察法

设计方法论：
- Design Thinking、双钻模型、用户旅程地图、信息架构设计、
  原子设计（Atomic Design）、响应式设计、无障碍设计（WCAG 2.1）

协作与交付：
- 设计系统搭建（Figma Component Library）、
  设计走查与开发协作、Jira/Confluence 项目管理
```

### 3.2 ATS 关键词覆盖

根据对 2025-2026 年 UI/UX 设计师 JD 的分析，以下关键词出现频率最高：

| 高频关键词（必须覆盖） | 中频关键词（尽量覆盖） |
|----------------------|---------------------|
| Figma、用户体验、交互设计 | 设计系统、组件库、响应式 |
| 用户研究、可用性测试 | Design Thinking、用户旅程 |
| 原型设计、高保真设计 | 信息架构、无障碍设计 |
| 视觉设计、设计规范 | 数据驱动设计、A/B 测试 |
| 跨部门协作、设计走查 | 动效设计、品牌设计 |

确保这些关键词自然出现在你的技能和项目描述中。

### Q: 设计师简历需要设计得很好看吗？

A: 需要区分场景。**在线投递**（经过 ATS 系统的投递渠道）时，必须使用 ATS 友好的标准格式简历——单栏、标准字体、可复制文字 PDF。**邮件直接发送给设计主管**时，可以使用设计感更强的排版，但仍需确保信息的可读性和结构清晰度。最佳策略是准备两个版本：一个 ATS 友好版用于在线投递，一个设计版用于邮件和面试现场。

### Q: 没有用户研究经验怎么办？

A: 用户研究不一定需要正式的项目经历。以下方式同样有效：
1. **个人练习项目**：选择一个你常用的 APP，进行一次完整的可用性测试（找 3-5 个朋友作为测试用户），记录问题并提出设计优化方案。
2. **课程项目深挖**：回顾你的设计课程项目，是否有用户调研的部分？重新整理并量化结论。
3. **竞品分析作为替代**：如果没有直接的用户研究经验，可以展示你的竞品分析能力——系统性地拆解竞品的设计模式，这也是设计决策能力的体现。

## 五、投递前的终极检查清单

- [ ] 作品集链接是否有效且可公开访问？
- [ ] 每个项目是否都包含 **设计决策的理由**（为什么这么设计），而非仅有视觉描述？
- [ ] 是否至少有一个项目展示了 **用户研究** 的过程（访谈、测试、数据分析）？
- [ ] 每个项目是否包含 **量化的结果**（转化率、满意度、效率提升等）？
- [ ] 技能部分是否覆盖了目标 JD 中的核心关键词（Figma、用户体验、交互设计等）？
- [ ] 简历是否为 **ATS 友好格式**（单栏、标准字体、无图片文字）？
- [ ] 是否准备了 ATS 版和设计版两套简历？
